Detailed Description

thread is the main object in kernel.

thread can be in one of the following states:

  • active
  • waiting (for sync object)
  • frozen
  • waiting frozen

After creation thread is in the frozen state

Thread is also specified by:

  • it's name
  • priority
  • stack size

HANDLE thread_create ( const char *  name,
int  stack_size,
unsigned int  priority,
THREAD_FUNCTION  fn,
void *  param 
)

creates thread object. By default, thread is frozen after creation

Parameters
name,:thread name
stack_size,:thread stack size in words. Must have enough space for context saving. Please, be carefull with this parameter: providing small stack may leave to overflows, providing large - to memory waste. Exact stack usage can be determined by turning on KERNEL_PROFILING.
priority,:thread priority. 0 - is highest priority, -idle- lowest. Only IDLE task can run on -idle- priority. Can be adjusted dynamically at any time.
fn,:thread function. Thread can't leave this function. In case of need of thread termination inside fn, thread_exit should be called.
param,:parameter for thread. Can be NULL
Return values
threadHANDLE on success, or INVALID_HANDLE on failure
